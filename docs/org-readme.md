# gh-toolkit org readme

Generate a profile README for a GitHub organization with LLM-powered descriptions.

## Usage

```bash
gh-toolkit org readme ORG_NAME [OPTIONS]
```

## Arguments

- `ORG_NAME` - GitHub organization name (required)

## Options

| Option | Type | Description | Default |
|--------|------|-------------|---------|
| `--output`, `-o` | PATH | Output file path | `README.md` |
| `--token`, `-t` | TEXT | GitHub personal access token | `$GITHUB_TOKEN` |
| `--anthropic-key` | TEXT | Anthropic API key for LLM descriptions | `$ANTHROPIC_API_KEY` |
| `--template` | CHOICE | Template style: `default`, `minimal`, `detailed` | `default` |
| `--group-by` | CHOICE | Group repos by: `category`, `language`, `topic` | `category` |
| `--stats/--no-stats` | FLAG | Include repository statistics | `--stats` |
| `--max-repos` | INT | Maximum repositories to include | All |
| `--exclude-forks/--include-forks` | FLAG | Exclude forked repositories | `--exclude-forks` |
| `--min-stars` | INT | Minimum stars required to include a repo | `0` |
| `--dry-run` | FLAG | Preview without writing to file | |
| `--help` | FLAG | Show help message | |

## Examples

### Basic Usage

```bash
# Generate README for an organization
gh-toolkit org readme my-org

# Preview without saving
gh-toolkit org readme my-org --dry-run

# Save to specific location
gh-toolkit org readme my-org --output profile/README.md
```

### Templates

```bash
# Default template (tables with stats)
gh-toolkit org readme my-org --template default

# Minimal template (simple list)
gh-toolkit org readme my-org --template minimal

# Detailed template (badges, extended info)
gh-toolkit org readme my-org --template detailed
```

### Grouping Options

```bash
# Group by inferred category (default)
gh-toolkit org readme my-org --group-by category

# Group by programming language
gh-toolkit org readme my-org --group-by language

# Group by first topic tag
gh-toolkit org readme my-org --group-by topic
```

### Filtering

```bash
# Only include repos with 5+ stars
gh-toolkit org readme my-org --min-stars 5

# Limit to top 20 repos
gh-toolkit org readme my-org --max-repos 20

# Include forked repositories
gh-toolkit org readme my-org --include-forks

# Combine filters
gh-toolkit org readme my-org --min-stars 10 --max-repos 15 --exclude-forks
```

## Output Format

### Default Template

```markdown
# Organization Name

A compelling tagline generated by LLM or derived from org description.

Mission statement about the organization's purpose.

## Focus Areas
- Python
- Web Development
- Machine Learning

## Repositories

### Libraries

| Repository | Description | Language | Stars |
|------------|-------------|----------|-------|
| [lib-name](url) | Description text... | Python | 42 |

## Stats
- **Repositories**: 25
- **Total Stars**: 150
- **Languages**: Python, JavaScript, TypeScript

---
*Generated with [gh-toolkit](https://github.com/michael-borck/gh-toolkit)*
```

### Minimal Template

Simple bullet list of repositories without tables or statistics.

### Detailed Template

Includes organization avatar, badges for each repository, language distribution charts, and extended metadata.

## LLM Integration

When an Anthropic API key is provided, the command uses Claude to generate:
- **Title**: A catchy 2-5 word title for the organization
- **Tagline**: A one-sentence description of what the org does
- **Focus Areas**: 3-5 key technologies or focus areas
- **Mission**: A 1-2 sentence mission statement

Without an API key, the command falls back to rule-based generation using the organization's existing description and top languages/topics.

```bash
# With LLM-powered descriptions
export ANTHROPIC_API_KEY=sk-ant-...
gh-toolkit org readme my-org

# Without LLM (uses fallback)
gh-toolkit org readme my-org
```

## Authentication

Requires a GitHub personal access token with `read:org` and `repo` scopes.

```bash
# Using environment variable
export GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxx
gh-toolkit org readme my-org

# Using command line option
gh-toolkit org readme my-org --token ghp_xxxxxxxxxxxxxxxxxxxx
```

## Category Inference

Repositories are automatically categorized based on:
- Topic tags (e.g., `library`, `cli`, `web-app`)
- Name patterns (e.g., `*-template`, `*-api`)
- Description keywords
- Primary programming language

Categories include:
- Libraries
- CLI Tools
- Web Applications
- APIs
- Templates
- Learning Resources
- Documentation
- Python/JavaScript/TypeScript Projects
- Other Projects

## Common Use Cases

### Organization Profile README

Create a README for your organization's `.github` repository:

```bash
gh-toolkit org readme my-org --output .github/profile/README.md
gh-toolkit org readme my-org --template detailed --output profile/README.md
```

### Quick Overview

Generate a minimal overview for internal documentation:

```bash
gh-toolkit org readme my-org --template minimal --no-stats
```

### Curated Showcase

Feature only your best repositories:

```bash
gh-toolkit org readme my-org --min-stars 10 --max-repos 10 --template detailed
```

## Error Handling

Common errors and solutions:

- **404 Not Found**: Organization doesn't exist or is private
- **401 Unauthorized**: Check your GitHub token
- **No repositories found**: Organization has no public repos matching filters
